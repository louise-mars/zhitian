# 知天 — 产品定位与差异化

## 一、为什么世界需要另一个天气 App

全球应用商店有超过 200 个天气应用。用户不缺天气数据——他们缺的是**基于天气的决策能力**。

当前天气 App 的根本问题：

| 问题 | 本质 |
|------|------|
| 告诉你"明天下雨"，但不告诉你该怎么办 | 信息 ≠ 决策 |
| 天气和日程是两个独立的 App | 用户需要手动关联 |
| 界面千篇一律，打开即关 | 没有情感连接，没有留存理由 |
| 广告满屏，隐私泄露 | 用户是产品，不是客户 |

**知天的回答：把天气从"被动查询工具"变成"主动决策助手"，同时用文化内容建立情感连接。**

---

## 二、产品定位

### 一句话定位

> **知天是一个将天气预报、日程管理与中国传统文化融为一体的智能生活助手。**

### 三层价值模型

```
┌─────────────────────────────────────────┐
│  第三层：情感价值（文化滋养）              │
│  每日诗词 · 农历节气 · 东方美学视觉        │
├─────────────────────────────────────────┤
│  第二层：决策价值（主动助手）              │
│  日程天气预警 · 今日宜忌 · 智能建议        │
├─────────────────────────────────────────┤
│  第一层：工具价值（精准数据）              │
│  15天预报 · 分钟级降雨 · 生活指数          │
└─────────────────────────────────────────┘
```

- **第一层**解决"天气是什么"——精准、可靠、快速
- **第二层**解决"我该怎么办"——天气×日程联动，主动推送决策建议
- **第三层**解决"为什么每天打开"——文化内容创造情感粘性

大多数天气 App 只做到第一层。知天的护城河在第二层和第三层。

## 三、核心差异化

### 差异化 1：日程天气预警引擎（AlertEngine）

**竞品现状**：天气 App 不知道你的日程，日历 App 不知道明天的天气。用户需要自己在两个 App 之间做关联判断。

**知天的做法**：

```
用户日程（App内 + 系统日历）
        ×
15天天气预报（逐日 + 逐小时）
        ×
气象灾害预警（气象局实时数据）
        ↓
智能预警："后天有客户拜访，预计大雨（降水18mm），建议带伞或调整出行时间"
```

技术细节：
- **室内事件过滤**：识别"Zoom会议"、"线上"等关键词，不对室内活动发预警
- **小时级精确匹配**：今天的定时事件检查 ±1 小时天气，避免"晚上暴雨但上午会议"的误报
- **多维度阈值**：降水量 >10mm、风速 >39km/h、温度 ≤0°C 或 >35°C
- **灾害预警关联**：气象局红/橙/黄预警自动关联今明两天的户外事件
- **本地推送**：明天/后天的预警通过 OS 通知主动触达，不需要打开 App

**这是市场上独一无二的功能。没有任何主流天气 App 做到天气×日程的深度融合。**

### 差异化 2：文化内容层（诗词系统）

**竞品现状**：天气 App 是纯工具，打开→看温度→关闭，平均使用时长 15 秒，无情感连接。

**知天的做法**：

- 400+ 首唐诗宋词，按天气/季节/节气智能匹配
- 一周内不重复，365 天持续新鲜感
- 点击展开完整全文，具有学习价值
- 暖金色衬线字体，东方美学视觉呈现
- 桌面 Widget 诗词模式——不打开 App 也能每天看到一句诗

**效果**：用户从"查天气"变成"看今天的诗"，打开频率和停留时长显著提升。

### 差异化 3：东方内敛美学

**竞品现状**：要么是 Material Design 标准模板，要么是过度渲染的 3D 动画（耗电）。

**知天的做法**：

- 宫崎骏风格天气动画（丁达尔光束、手绘感雨滴、萤火虫光点）
- 天气图标微动画（64dp Canvas，太阳脉冲/云漂移/雪花飘落）
- 背景渐变 700ms ease-in-out 平滑过渡
- 低电量/省电模式自动降级（不牺牲性能换视觉）
- 万年历风格日历（公历+农历+节气+节日+天气，五位一体）

**设计原则**：轻量、克制、高效。不做游戏化物理引擎，不做重度 3D 渲染。每一帧动画都有性能预算。

## 四、为什么用户需要知天

### 场景 1：职场人的早晨

> 7:00 起床，看一眼手机桌面 Widget：
> 「竹外桃花三两枝，春江水暖鸭先知」— 惠崇春江晚景·苏轼
>
> 打开 App：
> - ☀️ 23° 晴，体感 19°，风大体感偏冷
> - ⚠️ 后天有客户拜访，预计中雨（降水 15mm），建议带伞
> - 📅 今天：09:00 周会 / 14:30 外出拜访
>
> 一个界面，30 秒，今天和后天的决策全部完成。

### 场景 2：传统文化爱好者

> 每天打开 App 不是为了看天气，而是看今天的诗。
> 雨天推送"空山新雨后"，雪天推送"千山鸟飞绝"。
> 点击展开全文，读完一首完整的古诗。
> 收藏喜欢的，一年下来积累了自己的"诗集"。

### 场景 3：注重隐私的用户

> 不想用墨迹天气（广告太多），不想用彩云天气（数据上云）。
> 知天：无广告、无追踪、无后台偷跑、数据本地存储、开源可审计。

### 场景 4：在中国大陆使用 Pixel/三星等国际品牌手机的用户

> Google 天气不可用，苹果天气数据不准。
> 知天：高德定位（秒级）+ 和风天气（中国区最佳数据源）。

| **广告** | ❌ 无 | ✅ 大量 | ✅ 有 | ❌ 无 | ❌ 无 |

## 六、技术护城河

| 能力 | 实现难度 | 竞品复制成本 |
|------|---------|------------|
| 日程天气预警引擎 | 中 | 高（需要重新设计产品架构） |
| 400+ 诗词按天气/季节匹配 | 高（内容积累） | 高（需要文学素养+工程结合） |
| 室内事件智能过滤 | 低 | 低 |
| 小时级精确匹配 | 中 | 中 |
| 灾害预警×日程关联 | 中 | 高（需要打通多数据源） |
| 东方美学动画系统 | 中 | 中（设计能力依赖） |
| 动画性能降级 | 低 | 低 |

核心壁垒不在单个技术点，而在**产品理念的一致性**——天气、日程、文化三者的深度融合需要从架构层面重新设计，不是在现有天气 App 上"加个日历模块"能实现的。

## 八、总结

**知天不是又一个天气 App。**

它是一个回答"我该怎么办"的决策助手，一个每天用一句诗开启新一天的文化伴侣，一个尊重用户隐私和设备性能的工具。

用户不缺天气数据。他们缺的是：
1. 有人帮他们把天气和日程关联起来，主动提醒
2. 一个每天值得打开的理由（不是广告，是诗词）
3. 一个不偷数据、不耗电、不烦人的工具

**知天，知时节，知冷暖，知进退。**
